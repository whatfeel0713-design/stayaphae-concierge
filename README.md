# 압해 컨시어지 (stayaphae-concierge)

스테이 압해 예약 확정 고객 전용 컨시어지 웹앱. 설계 배경과 단계별 로드맵은
메인 리포(`staysoom`)의 `concierge-app-design.md`를 참고 — 이 리포는 그 문서의
Phase A(리포 뼈대 + 정적 콘텐츠 이식)와 Phase C(AI 챗)를 먼저 구현한 결과물이다
(순서상 Phase B보다 먼저 진행 — 아래 "왜 Phase B보다 먼저" 참고).

## 지금까지 한 것

**Phase A — 리포 뼈대 + 정적 콘텐츠 이식**
- Next.js 16 (App Router) + Tailwind v4 스캐폴드, 메인 사이트와 동일한 브랜드
  팔레트·폰트(Noto Sans/Serif KR) 적용.
- 메인 사이트 `/guide`의 정적 콘텐츠(이용 안내·오시는 길·로컬 맛집·추천
  코스·시크릿 쿠폰 안내)를 그대로 이식(`src/lib/guide-content.ts`).
- `verify_guide_access` RPC(메인 사이트와 동일 Supabase 프로젝트, anon key만
  사용)로 예약 코드를 검증하고, 성공 시 서명된 httpOnly 쿠키(JWT, `jose`)를
  발급해 이후 페이지 이동에 `?code=`가 계속 노출되지 않도록 함.
- `src/proxy.ts`에 쿠키 게이트 — `/` 이외의 라우트는 세션 쿠키 없으면
  `/`(코드 입력 화면)로 되돌린다. `/chat`, `/api/chat`이 이 게이트로 보호된다.

**Phase C — AI 컨시어지 챗 (`/chat`)**
- Claude API(`claude-opus-4-8`, 스트리밍) 기반 대화형 컨시어지.
  `src/app/api/chat/route.ts`가 세션 쿠키를 재검증한 뒤에만 응답 — 예약
  확정 손님만 API 비용이 드는 챗을 호출할 수 있다.
- **페르소나** (`src/lib/concierge-persona.ts`): 챗봇이 아니라 "이 집을 잘
  아는 사람"처럼 1인칭으로 답하도록 시스템 프롬프트를 설계. 이용
  안내·맛집·코스 전체 데이터를 근거로만 답하고, 모르는 것(정확한 일몰
  시각·실시간 날씨 등)은 지어내지 않고 문의 이메일로 안내하게 지시.
- **시간대 인지** (`src/lib/daypart.ts`): 지금이 KST 기준 새벽/아침/낮/오후/
  노을/저녁/밤 중 언제인지 시스템 프롬프트에 심어 자연스럽게 반영하고,
  `/chat` 진입 시 보여주는 추천 질문 칩도 시간대별로 다르게 구성(예: 노을
  시간엔 "노을이 가장 좋은 시간 알려줘", 저녁엔 "바베큐 준비 어떻게 해?").
- UI는 스트리밍 텍스트를 그대로 렌더링(마크다운 미변환 — 스트리밍 중 깨진
  태그 방지 모범 사례를 따름), 편집장풍 브랜드 톤(잉크/크림 팔레트) 유지.
- 비용/오남용 방어: 대화당 최대 30턴, 메시지당 최대 2000자로 클라이언트·
  서버 양쪽에서 제한. 로그 저장은 아직 없음(Phase B의 `concierge_logs`가
  생기면 `request_type='chat'`으로 붙일 수 있음).

**고도화 A그룹 — 무드 온보딩·사진 질문·T맵 전송·조용한 시간·갯벌 서사**
- **무드 온보딩**: `/chat` 첫 진입 시 "완전히 아무것도 안 하기 / 섬을 돌아보기
  / 둘만의 저녁" 3택(`src/lib/mood.ts`). 선택은 `localStorage`에 저장돼 다음
  방문에도 유지되고, 시스템 프롬프트 톤을 그 무드에 맞춘다.
- **사진으로 물어보기**: 챗 입력창에 사진 첨부 버튼 — 커피머신·보일러 등을
  찍어 보내면 Claude Vision으로 바로 답한다(`/api/chat`이 이미지 콘텐츠
  블록을 받아 처리, 대화 기록에는 이미지를 남기지 않고 텍스트만 유지해
  비용을 관리).
- **T맵 원클릭 전송**: 답변에 언급된 장소(섬 이름·맛집 등)를 감지하면
  버튼으로 T맵 딥링크를 띄운다(`src/lib/place-names.ts`).
- **조용한 시간 넛지**: 밤·새벽 시간대에 홈 화면·챗 화면 모두에서 마당
  소음 자제 안내 배너를 보여준다(`daypart.ts` 기반).
- **갯벌 스토리텔링**: 신안 갯벌이 2021년 유네스코 세계자연유산에 등재됐다는
  사실을 페르소나와 추천 코스 문구에 반영.

**신안 특산품·축제 정보**
- **호스트 직판 상품**(`src/lib/local-products.ts`): 천일염·무화과를 챗과
  가이드 데이터에 반영 — 실제 가격·재고는 플레이스홀더, 문의로 조율.
- **신안·목포 축제**(`src/lib/festivals.ts`): 축제 날짜는 해마다 바뀌어
  하드코딩하면 금방 틀린 정보가 되므로, 계절 개요만 고정 정보로 두고
  Claude의 `web_search` 서버 도구(`web_search_20260209`)를 `/api/chat`에
  연결해 실시간으로 정확한 날짜를 찾아 답하게 했다(턴당 최대 3회 검색으로
  비용 제한).

**고도화 B그룹 — 실시간 날씨·물때 연동**
- `src/lib/weather-tide.ts` + `src/app/api/chat/route.ts`: `get_weather_forecast`
  / `get_tide_info` 커스텀 도구를 챗에 연결. 지금까지는 "노을이 드는 시간"
  처럼 계절감으로만 답하던 것을, 기상청 단기예보(진짜 API, 신뢰도 높음)와
  국립해양조사원 조위 정보(만조/간조 시각)로 실제 정확하게 답할 수 있다.
- **도구 사용 루프**: 웹서치(서버 도구)와 달리 날씨·물때는 우리 서버가 직접
  호출하는 커스텀 도구라, `/api/chat`이 스트리밍하면서 `tool_use` 정지
  사유를 감지해 도구를 실행하고 결과를 이어붙여 재요청하는 루프를 최대
  6회까지 돈다.
- **설정 필요**: `KMA_SERVICE_KEY`/`KMA_NX`/`KMA_NY`(기상청), `KHOA_SERVICE_KEY`/
  `KHOA_OBS_CODE`(국립해양조사원) — 비어 있으면 도구가 "아직 설정되지
  않았다"고 솔직히 답한다(지어내지 않음). ⚠️ 조위 API는 이번 작업 세션에서
  웹 검색이 일시 장애라 최신 스펙을 재확인하지 못한 채 구현했다 — 서비스키
  발급 후 반드시 실제 응답을 한 번 확인하고 필요하면 파싱을 조정할 것
  (`weather-tide.ts` 상단 주석 참고). 기상청 API는 안정적인 공공 API라
  신뢰도가 높다.

**고도화 B그룹 — 개인화된 첫 인사·시크릿 쿠폰 실물화**
- **개인화된 첫 인사**: 메인 리포(`staysoom`)에 새 RPC
  `get_guide_session_info`를 추가(마이그레이션
  `20260721090000_guide_session_info.sql`, ⚠️ Supabase SQL Editor에서 직접
  실행해야 적용됨). 예약자 이름·인원·체크아웃 날짜·기념일 메모(신규
  `special_occasion` 컬럼, `/admin/reservations`에서 입력)를 반환한다.
  이 앱은 코드 인증 성공 시 이 정보를 세션 쿠키(JWT)에 담아 홈 화면
  ("OOO님, 어서오세요")과 챗 시스템 프롬프트(대화 시작 시 한 번만 이름
  호명, 기념일이면 서프라이즈 제안)에 반영한다. RPC가 아직 적용되지
  않았거나 실패해도 `verifyGuideAccess`만으로 정상 동작하도록 우아하게
  폴백(`src/lib/guide-access.ts`).
- **세션 TTL 정교화**: 체크아웃 날짜를 알 수 있으면 그날 자정(KST) +
  하루 여유로 쿠키 만료를 맞추고, 모르면 기존 5일 고정 TTL로 폴백한다
  (`src/lib/session.ts`).
- **시크릿 쿠폰 실물화**: `reveal_secret_coupon` 커스텀 도구 — 손님이
  요청하면 QR 코드를 화면에 띄운다(`src/lib/secret-coupon.ts`, `qrcode`
  라이브러리). `concierge_logs`(Phase B)가 아직 없어 발급 이력은 남기지
  않고, 예약 코드+날짜로 결정론적 코드를 생성해 같은 날엔 같은 코드가
  나오게 한다. 도구 실행 결과(QR 이미지)는 Claude의 텍스트 스트림과는
  별도로 NUL 문자로 감싼 아웃오브밴드 마커를 통해 클라이언트로 전달되고,
  `chat-client.tsx`가 이를 파싱해 카드 UI로 렌더링한다(모델의 컨텍스트에
  이미지를 태우지 않아 토큰 비용 절감).

**Phase 1 확장 준비 — 기기 사용법**: 사용자가 나중에 핸드드립·에어컨·
커피머신·와이파이 연결·다도·블루투스 스피커·씨네빔 큐브 빔프로젝터·넷플릭스
연결 등의 실제 사용법을 알려주면 바로 반영할 수 있도록 `src/lib/
device-manuals.ts`를 미리 준비해 뒀다. 각 항목은 `ready: false`로 시작하고,
챗은 이런 항목을 물으면 지어내지 않고 "정리 중이니 웰컴 카드나 문의로
확인해 달라"고 솔직히 답한다. **채우는 법**: 해당 항목의 `ready`를 `true`로,
`steps` 배열에 순서대로 문장을 넣으면 챗 시스템 프롬프트에 자동 반영된다.
아직 정적 페이지(`page.tsx`)에는 노출하지 않는다 — "준비 중" 카드로 가득한
섹션은 방문객에게 좋은 인상을 주지 않으므로, 실제 내용이 채워진 뒤 페이지
섹션 추가를 검토할 것.

**고도화 C그룹 — 여행 사진 캡션·퇴실 후 편지**
- **여행 사진 감성 캡션**: 기존 "사진으로 물어보기"(A그룹) 인프라 위에,
  손님이 사진과 함께 캡션·SNS 문구를 요청하면 담백한 한 줄(20자 내외)을
  제안하도록 페르소나에 지시만 추가했다 — 새 인프라 없이 프롬프트만으로
  구현되는 저비용 기능.
- **퇴실 후 감성 여정 편지**: 이 앱이 아니라 **메인 리포(`staysoom`)**에
  구현했다 — 게스트 이메일은 메인 리포의 `reservations` 테이블에만 있고
  (개인정보 최소 노출 원칙상 이 앱의 세션 쿠키에는 담지 않음), 발송 트리거도
  관리자가 `/admin/reservations`에서 누르는 방식이 자연스럽기 때문이다.
  `src/lib/notifications/checkout-letter.ts`(Resend REST API) 참고. ⚠️ 백로그
  원안은 "이번 스테이 동안 물은 질문·언급한 장소" 기반 개인화였지만,
  `concierge_logs`(Phase B)가 없어 대화 기록을 저장하지 않으므로 지금은
  예약 정보(이름·날짜·기념일)만으로 따뜻한 정적 편지를 보낸다 — 대화 기반
  개인화는 챗 로그 저장이 생긴 뒤의 다음 단계.

### 왜 Phase B보다 먼저 Phase C를 했는지

설계 문서(`concierge-app-design.md`)의 원래 순서는 B(서비스 신청)→C(AI 챗)다.
하지만 AI 컨시어지 자체가 "이 앱의 특별함"을 가장 잘 드러내는 기능이라
판단해 먼저 구현했다. `concierge_logs` 테이블(Phase B)이 아직 없어 챗 로그는
저장되지 않는다 — Phase B를 진행하면서 `request_type='chat'`으로 로깅을
추가하고, 바베큐 신청도 챗 안에서 자연스럽게 요청하면 컨시어지가 안내하는
흐름으로 확장할 수 있다.

## 알려진 단순화 (다음 단계에서 정교화 필요)

- **세션 만료**: `verify_guide_access` RPC가 boolean만 반환해 정확한 체크아웃
  날짜를 알 수 없다. 지금은 고정 TTL(5일)로 쿠키를 발급한다. 체크아웃 자정
  만료로 정교화하려면 메인 리포 쪽에 RPC가 checkout을 함께 반환하도록 하는
  마이그레이션이 필요하다(`src/lib/session.ts` 주석 참고).
- **홈 화면의 체크인/체크아웃 D-day**: 설계 문서 IA표에 있는 항목이지만 같은
  이유(RPC가 예약 상세를 반환하지 않음)로 이번 단계에서는 넣지 않았다.
- **챗 대화 로그 미저장**: 위 설명대로 `concierge_logs`가 생기기 전까지는
  대화가 서버에 남지 않는다(클라이언트 상태로만 존재, 새로고침 시 사라짐).
- **비용**: `claude-opus-4-8`은 Opus 등급 중 가장 비용이 크다. 트래픽이
  늘어나면 `src/app/api/chat/route.ts`의 `model`을 `claude-sonnet-5` 등으로
  낮추는 것을 고려할 것 — 지금은 응답 품질을 우선해 Opus로 시작한다.
- **무드 선택의 `localStorage` 읽기**: 서버 렌더링 시점엔 `localStorage`를
  알 수 없어 클라이언트 하이드레이션 직후 무드 픽커 표시 여부가 한 프레임
  바뀔 수 있다(개발자 콘솔에 하이드레이션 경고가 뜰 수 있음) — 기능에는
  영향 없는 사소한 이슈.

## 로컬 개발

```bash
npm install
cp .env.local.example .env.local   # 값 채우기 — 메인 사이트와 같은 Supabase 프로젝트
npm run dev
```

`GUIDE_SESSION_SECRET`은 메인 사이트의 어떤 값과도 무관한 이 앱 전용 서명
비밀값이다 (`openssl rand -hex 32`로 생성). `ANTHROPIC_API_KEY`가 없어도
앱은 빌드·구동되지만 `/chat`의 응답은 503을 반환한다.
